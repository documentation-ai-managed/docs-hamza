## Overview

Hamza provides a centralized space to organize, edit, and publish your project documentation. You create structured pages using MDX, collaborate with teams, and maintain version history effortlessly. Benefit from intuitive navigation, search, and customizable themes to streamline your workflow.

Hamza offers powerful tools for documentation management.

## Quick Start

Get started with Hamza in minutes.

<Steps>
  <Step title="Create Your Space" icon="plus">
    Sign up at `https://hamza.example.com` and create a new documentation space for your project.
  </Step>
  <Step title="Add Your First Page" icon="file-plus">
    Click **New Page**, choose `introduction.mdx`, and start editing with the visual editor.
  </Step>
  <Step title="Publish Changes" icon="upload">
    Preview your page, then hit **Publish** to make it live for your team.
  </Step>
</Steps>

<Callout kind="tip">
  Customize your space theme using the brand color `#3B82F6` in settings for a consistent look.
</Callout>

## Brand Guidelines

Follow these guidelines to maintain consistency.

<Tabs>
  <Tab title="Primary Color" icon="palette">
    Use `#3B82F6` as your primary brand color.

    ```css
    :root {
      --brand-primary: #3B82F6;
    }

    .button {
      background-color: var(--brand-primary);
      color: white;
    }
    ```
  </Tab>
  <Tab title="Typography" icon="type">
    Pair with sans-serif fonts for readability.

    <CodeGroup tabs="CSS,MDX">
    ````css
    body {
      font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
    }
    ````
    ````mdx
    ## Heading (H2)

    Use markdown headings for structure.
    ````
    </CodeGroup>
  </Tab>
</Tabs>

<Expandable title="Advanced Customization" default-open="false">
  Override default styles with custom CSS variables. Inject via space settings:

  ```javascript
  // Example config
  const config = {
    theme: {
      colors: {
        primary: '#3B82F6'
      }
    }
  };
  ```

  Test changes in preview mode before publishing.
</Expandable>
